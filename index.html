STL Coders "Talk is cheap. Show me the code."

README.TXT

This is the first piece of code written for StlCoders. The source can be found at GitHub.

We're two developers who talked about code at work and decided to start a coder group. Here's some guidelines:

More code, less talk

Our meetings will focus on writing actual code. Which sounds more fun to you?

  1. Watch a presentation
  2. Take notes about history, setup, and syntax of a language or tool
  3. See a few examples
  4. Listen to Questions and Discussion
  5. Go home and throw your notes on a pile
Or,
  1. Sit with 2-3 fellow coders at an actual computer
  2. Write code
  3. Publish the code
(We're in the 2nd group, by the way)

Frequent Meetings

We plan to meet *weekly* and very informally. Beer and coffee are requisites, as well as laid-back atmosphere. The less time that's spent preparing for a meeting is more time that can be spent writing actual code.

Pair programming, beginner/n00b friendly

Coders should take turns at the keyboard. The purpose is to learn and to write code. You can't write code by watching.

Share what we do.

We intend to post snippets of what we've done to GitHub. Perhaps the stuff we work on will spur more contributions and other ideas.

Enough talk. No, I won't keep you updated. If you're interested, e-mail me at nate@nateneff.com
